# ADR 0001: Record Architecture Decisions

## Status
Proposed

## Context
We need a way to record and track architectural decisions for the Trader project.

## Decision
We will use Architectural Decision Records (ADRs) stored in the `docs/decisions/` directory as Markdown files.

## Consequences
- Better transparency into why decisions were made.
- Easier onboarding for new contributors.
- Historical record of the project's evolution.
